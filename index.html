Real-time Data Ingestion & Transformation Platform

Architected and implemented a robust, highly-available data ingestion and transformation platform critical for Act-On's marketing automation services. This system processes two modes of data, batch upload and streaming. The batch upload mode processes one million records with 52 columns (1 GB of data) in less than one minute while generating a verbose error report for any record that fails validation, empowering the user.

    Core Technologies: Leveraged Apache Kafka for high-throughput message queuing, AWS Snowpipe for efficient streaming data ingestion into Snowflake, and Spring Cloud Data Flow for orchestrating complex data transformation pipelines.

    Scalability & Performance: Designed the system to dynamically scale with increasing data volumes, consistently maintaining sub-second latency for critical data pathways. Optimized Snowflake queries and data models to ensure efficient processing and retrieval.

    Impact: Directly enabled new product features requiring immediate data availability, improved the power of segmentation, and reduced data processing costs by optimizing resource utilization.

Legacy Service Modernization & Cloud Migration

Led the comprehensive modernization of a mission-critical, monolithic dataflow service responsible for core marketing automation logic. This initiative was key to improving system stability, performance, and laying groundwork for full cloud migration. This system consumed 7 different sources of data and used kinesis streams to send the data to 13 different downstream dependencies, including various microservices, Snowflake, and S3.

    Technical Upgrade: Orchestrated the upgrade of the entire service codebase from Java 8 to Java 21, resolving numerous compatibility issues and leveraging modern language features for improved performance and maintainability. This also involved upgrading our H2 instances, which required careful collaboration with our Ops team.

    Database Centralization: Consolidated disparate, distributed H2 databases into a centralized PostgreSQL system, eliminating data inconsistencies and simplifying database management.

    Migration Strategy: Developed a detailed rollout plan and comprehensive test suite to mitigate risk during the upgrade and database migration, ensuring zero downtime for customers.

    Cloud Readiness: This modernization significantly improved service visibility and reduced its footprint, directly enabling a seamless migration of our infrastructure from an Flexential data centers to AWS, improving overall operational efficiency and resilience.

Data Lake Performance & Reliability Enhancement

Identified and resolved critical stability and performance issues within Act-On's core data lake query service, transforming it from a bottleneck into a highly reliable and efficient data foundation.

    Problem Identification: Diagnosed root causes of severe data errors (averaging 800 per hour) and unacceptably slow response times (up to 8 minutes) affecting core customer analytics.

    Solution Implementation: Implemented a Snowflake deferred merge strategy and optimized data ingestion techniques, drastically reducing data loss and eliminating data duplication.
